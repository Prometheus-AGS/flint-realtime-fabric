## Technical Debt Introduced

1. **wasm-opt skipped (`--no-opt`)** — The wasm-bindgen-cli 0.13.1 ships an older wasm-opt that fails bulk-memory validation (introduced by Loro CRDT). The `--no-opt` flag produces a larger `.wasm` binary (~2x). Debt: upgrade wasm-opt to 0.116+ in Dagger Stage 6 to restore size optimization.

2. **Kotlin JNI test disabled** — Gradle `tasks.withType<Test>` is not disabled yet; the `compileKotlin` task passes but runtime JNI tests would fail without `libfrf_ffi.so`. Debt: add `tasks.withType<Test> { enabled = false }` to `lib/build.gradle.kts` for CI correctness.

3. **Layer 2 E2E CRDT button test absent** — The `signaling.spec.ts` CRDT button test is Layer 1 only. A Layer 2 test that exercises the CRDT apply-delta path through the WASM module was assessed but not implemented (requires full compose stack + real WASM). Debt: add `layer2-crdt.spec.ts` in a future phase.

4. **`frf.kt` gitignored** — The `.gitignore` in `sdks/kotlin/` lists `frf.kt` as gitignored (regenerated by `build_jni.sh`). The file currently exists in the repo. Either remove it from `.gitignore` (to commit the generated binding) or delete it from the tree. Both are valid; current state is inconsistent.

5. **Compose keto config** — `deploy/keto/keto.yml` and `deploy/oathkeeper/config.yml` exist as placeholders. The compose smoke passed because keto healthcheck uses `wget` which succeeds on a running container regardless of config validity. A deeper keto integration test is deferred.

## Lessons Captured

- **wasm-opt bulk-memory**: UniFFI-generated WASM with Loro uses bulk-memory ops. wasm-bindgen-cli ships its own wasm-opt; the system wasm-opt may be incompatible. The `--no-opt` bypass is valid until wasm-bindgen-cli >= 0.2.100 or a standalone wasm-opt >= 0.116 is injected.

- **`bash` parameter expansion `/${pattern}/` pitfall**: In bash `${var/pattern/replacement}`, slashes inside `pattern` do NOT need escaping — writing `\/` is treated as the literal two characters `\/`, not `/`. Use `sed 's|/main/|/new/|'` for path segment replacement to avoid this silent failure mode.

- **Kotlin wrapper bootstrap**: `gradle wrapper` must be run from the module directory that has `settings.gradle.kts`. Running from the workspace root will write the wrapper jar to the workspace root, not the module. Always `cd` first.

- **ktlint + UniFFI**: UniFFI-generated `.kt` files violate `filename`, `function-naming`, `no-wildcard-imports`, and `property-naming` rules by design (JNI naming conventions). The correct fix is a scoped `.editorconfig` block that disables these rules for the generated file path — not suppression annotations in the generated file.

- **SurrealDB v3 breaking changes**: `file://` is deprecated; `surrealkv://` is required. The `--auth` flag was removed. The `surreal` binary is not on PATH in the Docker image; healthcheck must use `/surreal is-ready`.

- **Docker runtime image glibc**: `rust:latest` is Debian 13 (trixie, GLIBC 2.41). The runtime image must match — `debian:trixie-slim`. Using `debian:bookworm-slim` (Debian 12, GLIBC 2.36) causes `GLIBC_2.38 not found` for the compiled binary.
